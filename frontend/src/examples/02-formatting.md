# Text formatting

Inline styling is plain Markdown — type the markers or use the toolbar / shortcuts.

| You write | You get | Shortcut |
| --- | --- | --- |
| `**bold**` | **bold** | ⌘B |
| `_italic_` | _italic_ | ⌘I |
| `~~strikethrough~~` | ~~strikethrough~~ | — |
| `` `inline code` `` | `inline code` | ⌘E |
| `[a link](https://example.com)` | [a link](https://example.com) | ⌘K |

## Combining styles

You can nest emphasis: **bold with _italic_ inside**, or mark some **`code in bold`**. Links can wrap formatted text too, like [**this bold link**](https://example.com).

## Links

A link renders as just its text — the URL stays out of the way until your cursor enters it. **⌘-click** (or Ctrl-click) any link to open it in your browser:

- Inline: [MarkdownMD on GitHub](https://github.com/emrul/md)
- Autolinks and reference-style links round-trip cleanly too.

## Escaping

Need a literal asterisk or underscore? Backslash-escape it: \*not italic\* and \_not italic\_. Inline code is the simplest way to show Markdown verbatim, like `**this stays literal**`.
